Guideline: Scrum of Scrums
Main Description

A principal forma de escalar o Scrum, para trabalhar com grandes times é coordenar um "Scrum de Scrums". Com essa abordagem cada Scrum Team atua normalmente, mas cada time também contribui com uma pessoa para assistir às Reuniões Scrum de Scrum e coordenar o trabalho de múltiplos Scrum Teams. Essas reuniões são análogas às Reuniões Daily Scrum, mas não necessariamente acontecem todo dia. Em muitas organizações é suficiente ter uma Reunião Scrum de Scrums de duas a três vezes por semana.

A ilustração abaixo mostra como uma abordagem Scrum de Scrums permite o escalamento (neste caso para 243 pessoas). Cada célula representa uma pessoa em um Scrum Time. A parte de baixo desta ilustração mostra times com nove desenvolvedores em cada um. Uma pessoa de cada time (a célula de cor diferente) também participa em um Scrum de Scrums para coordenar o trabalho acima daquele time. Então, daquelas nove pessoas do time, uma é selecionada (desta vez assinalado com linhas diagonais) para participar no que poderia ser chamado de Scrum de Scrums de Scrums.
